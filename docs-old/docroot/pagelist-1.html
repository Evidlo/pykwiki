Frequently Asked Questions

About authoring

How do I specify the date of the page?

In the page configuration section, set the timestamp key. It must match the format from config.yaml - timestamp_format.

Example

[[
title: My Page
tags: [stuff, moar stuff]
timestamp: 2013-11-13 21:14
]]

See: Page Data Block for more information.

How do I set a list of posts as my homepage?

To set your home page to the /posts.html url, change config.yaml -> home_page to the following:

...
home_page: posts.html
...

About configuring

What does a project directory look like?

You should checkout Getting Started, but it basically looks like this:

MyProject/
    links.yaml
    config.yaml
    source/
    docroot/
    themes/

What webservers work with PyKwiki?

Any webserver that serves static files should work with PyKwiki. Two popular choices include Apache2 and Nginx.

What versions of Python work with PyKwiki?

PyKwiki officially works with the following versions:

  • Python == 2.7
  • Python >= 3.3

Python 3.2 is not supported because the Jinja2 2.6 package does not support Python 3.2. Because PyKwiki requires Jinja2, there is no way around this.

Application Programming Interface

conf Object

The conf object is made available to all theme templates.

Properties

config.yaml

Any property in config.yaml is available to this object.

Useful properties for themes

  • web_prefix - When a theme makes calls to local resources, it should join the path to the resource with this property. $.get('{{conf.web_prefix}}/idx.json') for example.
  • search_tpl - The search template. Default is "search.html".

ctrl Object

The ctrl object contains information about all PyKwiki pages on a site. It has several useful methods for generating lists of pages and calendars.

Functions

  • get_pages()
    • sort_key='mtime' - The page object property to sort the returned pages by.
    • direction='descending' - The sort order. descending or ascending.
    • limit=None - The maximum number of pages to return. 0 or None is unlimited.
    • filters=None - A nested list of filters and operators. Example:
      • [['mtime','>',1234567890], ['title','!=','Foo']] will filter on pages that do not have the title of "Foo" and have a date greater than the epoch of "1234567890".
    • private=False - Bool that indicates whether or not the results should include pages marked as private. Default is false.

page Object

The page object contains all information about a specific page. It is made available to the page.html template as page.

Properties

General information

  • title - The page's title from the page data block.
  • blurb - The blurb or description of the page. This will be auto generated on access if no blurb or description was specified in the page data block.
  • author - The author of the page if specified, else returns the site's author.
  • description - The page's description; used in <meta name="description">.
  • keywords - A string of keywords; used in <meta name="keywords">.
  • tags - A list of tags for the page.
  • url - The relative url to access the page. Includes web_prefix if specified.
  • source_path - The full file system path to the source page.
  • target_path - The full file system path to the rendered (docroot) page.

Content

  • source_text - The raw markdown of the page.
  • target_text - The rendered HTML of the page.

Date and time

  • mtime - Epoch of the page's timestamp if specified, else it's the modified date.
  • mtime_string - String of the page's time, formatted by conf.time_format.
  • mdate_string - String of the page's date, formatted by conf.date_format.
  • mtimestamp - String of the page's timestamp, formatted by conf.timestamp_format.
  • mtime_tuple - A time.localtime() time struct.
  • mtime_hour - The (24) hour of the page timestamp.
  • mtime_minute - The minute of the page timestamp.
  • mdate_day - The day of the month of the page timestamp.
  • mdate_month - The month of the page timestamp.
  • mdate_year - The four digit year of the page timestamp.
